# 时间线轨道拖拽 & 落位交互规范

> 对齐剪映标准 · TimelineKit 正式交互文档
>
> 所有轨道拖拽、排序、落位相关功能的开发，必须遵守本文档规则。

---

## 一、核心总体原则

| # | 原则 |
|---|------|
| 1 | 轨道严格分为两类：**主轨道**（`isMainTrack = true`）和**附属轨道**（字幕/文字/贴纸/音效等）。 |
| 2 | **同一轨道内部**：所有片段严禁重叠、严禁时间交叉。 |
| 3 | **不同轨道之间**：完全允许重叠、自由摆放，无任何限制。 |
| 4 | 主轨道与附属轨道采用**两套完全独立**的拖拽 / 排序 / 落位逻辑，不能共用。 |

---

## 二、主轨道专属交互规则（`isMainTrack = true`）

1. 轨道内片段默认**无缝连续排布**：无空隙、不重叠。
2. 支持**长按进入紧凑重排模式**，可拖动片段互相交换位置、重排顺序。
3. 交换后自动保持整体连续，总时长不变，自动衔接前后片段。
4. 「交换重排、无缝吸附」能力**仅属于主轨道**，附属轨道严禁开放。

---

## 三、附属轨道统一规则（字幕 / 文字 / 贴纸 / 音效等非主轨道）

### 3.1 彻底关闭片段交换 / 重排逻辑

附属轨道**不允许**长按触发紧凑重排模式，不允许两个片段互相换位。

### 3.2 仅支持两种基础操作

| 操作 | 说明 |
|------|------|
| **整体平移** | 左右拖动片段，改变其整体开始时间（`targetRange.start`） |
| **边缘拉伸** | 拖动左/右裁切柄，调整时长（`targetRange.duration`） |

### 3.3 同轨道防重叠 & 防硬塞规则

拖动片段在同轨道内平移时，实时计算与相邻片段的间距：

```
设 draggingDuration = 拖拽中片段的时长
设 gap              = 目标落位空隙的时间宽度

if gap >= draggingDuration  → 允许落位（边缘可吸附对齐）
if gap <  draggingDuration  → 禁止落位，拖拽自动卡位，片段弹回最近合法位置
```

**任何情况下，同轨道片段不能出现时间重叠或边缘交叉。**

### 3.4 跨轨道自由规则

将片段拖到另一行轨道时（跨轨道），不受任何时间重叠限制，可与上层轨道的画面 / 时间段自由叠加摆放。

> 注：跨轨道拖拽为后续功能，当前版本暂不实现，但设计时不得引入阻止跨轨道重叠的逻辑。

---

## 四、防重叠计算方式（同轨道平移）

对同轨道内除自身以外的所有片段，计算允许的平移范围：

```
// 找左侧最近的邻居
leftNeighborEnd   = max(seg.end for seg in track.segments where seg != dragging && seg.end <= dragging.start)
                    默认为 0（时间轴起点）

// 找右侧最近的邻居
rightNeighborStart = min(seg.start for seg in track.segments where seg != dragging && seg.start >= dragging.end)
                     默认为 +∞（无限制）

// 合法起始时间范围
minStart = leftNeighborEnd
maxStart = rightNeighborStart - dragging.duration

// 拖拽时将 newStart 钳制在 [minStart, maxStart]
newStart = clamp(newStart, minStart, maxStart)
```

若 `maxStart < minStart`（即空隙宽度 < 片段时长），片段无法进入该空隙，钳制后自动停在边界。

## 五、当前已知问题（待修复）

| # | 问题描述 | 根因 |
|---|---------|------|
| 1 | 附属轨道套用了主轨道的「长按交换重排」逻辑，片段换位后时间区间错乱重叠 | `buildSegments()` 未按 `isMainTrack` 过滤长按手势 |
| 2 | 自由平移时缺少同轨道空隙宽度校验，长片段可硬塞进窄空隙 | `handleFreePan` 只做了 `max(0, newStart)` 下界钳制，未做右侧邻居钳制 |
| 3 | 附属轨道拥有不该有的换位重排能力 | 同上，手势未隔离 |

## 六、开发确认项

以下各条请开发逐条确认可行性后再动工：

1. **是否可以按主轨道/附属轨道拆分两套独立拖拽逻辑？**
   → 已有 `isMainTrack` 字段，从 `buildSegments()` 处分支即可，无架构障碍。

2. **是否可以禁用所有附属轨道的片段交换、重排插队能力？**
   → 只需在 `buildSegments()` 内对 `!track.isMainTrack` 的 block 不注册 `onLongPressActivated` 回调，长按手势仍可保留（用于未来跨轨道拖动），但不触发紧凑重排模式。

3. **是否可以实现同轨道「片段防重叠 + 空隙宽度校验，禁止硬塞入窄空隙」？**
   → 在 `handleFreePan(.changed/.ended)` 中，根据第四节公式实时计算 `[minStart, maxStart]` 并钳制 `newStart`，技术上完全可行。

4. **是否可以支持跨轨道允许自由重叠、同轨道严格禁止重叠的规则？**
   → 防重叠逻辑只作用于「同一 `TrackRowView` 内」，不跨行计算，天然满足跨轨道无限制的要求。
