# 片段手柄交互规范

> 对齐剪映标准 · TimelineKit 正式交互文档  V1.1
>
> 适用范围：所有轨道所有片段（视频/图片/字幕/贴纸/文本/音效）
>
> 本文档替代旧版 `trim-handle-boundary-spec.md`，所有手柄相关开发以本文为准。

---

## 一、手柄显示规则

### 1.1 主轨道（`isMainTrack = true`）

- **默认常驻显示**左右拉伸手柄，无需选中，随时可拉伸修剪。
- 符合主轨道高频剪辑习惯。

### 1.2 附属轨道（字幕 / 文字 / 贴纸 / 音效等）

| 状态 | 手柄 |
|------|------|
| 未选中 | **完全隐藏**，界面干净 |
| 选中 | 弹出左右手柄，视觉加宽 + 热区放大 |
| 取消选中 | 手柄立刻隐藏 |

### 1.3 点击选中行为

| 触发 | 效果 |
|------|------|
| 点击**主轨道**片段 | 选中 + 弹出编辑面板 |
| 点击**附属轨道**片段 | 仅选中高亮，**不弹出编辑面板** |
| 点击空白处 | 取消所有选中，所有附属手柄隐藏 |

> 内容编辑（改文字/样式）统一从底部工具栏入口进入，不干扰时间线操作。

## 二、手柄触摸规格

### 2.1 尺寸规格

| 状态 | 视觉宽度 | 触摸热区宽度 |
|------|---------|------------|
| 正常（主轨道常驻） | 6 pt | 24 pt |
| 选中（附属轨道选中后） | 9 pt | 32 pt |

热区向片段**内侧**扩展，覆盖范围内的触摸均触发拉伸。

### 2.2 手势隔离

```
┌────────────────────────────────────────┐
│  [← 热区]     片段主体区域     [热区 →]  │
│  触发拉伸 ←──────────────────→ 触发拉伸 │
│             触发长按平移                │
└────────────────────────────────────────┘
```

- 触摸落在手柄热区（子视图）→ 触发拉伸，长按自动取消
- 触摸落在主体区域（父视图）→ 触发长按平移，手柄不响应
- 基于 UIKit subview hitTest 自然隔离，无需额外 delegate 协调

## 三、手柄拉伸防覆盖规则（核心铁律）

**同一轨道内，拉伸手柄绝不允许侵入相邻片段的时间区间。**

### 3.1 左手柄（调整入点）

```
[ 前一片段 ] ─── 空隙 ───[ 当前片段 |← 左手柄 ]
                          ↑
                  最多贴到前一片段的右边缘（end）
                  绝对不能越过该边缘向左侵入
```

约束公式：

```
newStart ≥ max(0, prevSibling.end)
newStart ≤ trimStartRange.end - minDuration (保留最小时长)
```

若该侧无相邻片段，则左边界为 `0`（时间轴起点）。

#### 3.1.1 附属轨道左手柄：素材约束 vs. 无约束

| 片段类型 | 左手柄左边界 | 说明 |
|---------|------------|------|
| 文字 / 字幕 / 叠加 / 调节 | `max(0, prevSibling.end)` | 无素材时长限制，仅受相邻片段阻挡 |
| 音频（附属轨道） | `max(prevSibling.end, trimStartRange.start - sourceRangeStart)` | 受源音频文件可用预卷限制，不能超出素材开头 |
| 视频（主轨道） | 预卷消费模型，见 section 3.1.2 | start 锚定，右边缘向右扩展 |

- 文字/字幕/叠加/调节没有源素材概念，可自由向左拉伸到轨道起点或前一片段右边缘。
- 音频的 `sourceRangeStart` 表示源文件中当前入点之前的可用秒数，左手柄最多向左拉伸该值。
- `sourceRangeStart` 为 `nil` 或 `0` 时无预卷可用，左手柄被钉死在当前入点（无素材可扩展）。

### 3.2 右手柄（调整出点）

```
[ 当前片段  右手柄 →| ] ─── 空隙 ─── [ 后一片段 ]
                    ↑
            最多贴到后一片段的左边缘（start）
            绝对不能越过该边缘向右侵入
```

约束公式：

```
newEnd ≤ nextSibling.start
newEnd ≥ trimStartRange.start + minDuration (保留最小时长)
```

若该侧无相邻片段，则右边界为 `+∞`（可无限向右延伸）。

### 3.3 实时卡位

- 边界约束在手指拖动过程中**实时生效**（`.changed` 事件）
- 不等松手再校验，拖到边缘即刻卡死，无法再往里拖
- 松手后落点已是合法位置，直接提交，无需回弹

### 3.4 边界计算时机

- 在 `.began` 时根据当前 `siblingRanges` 计算并缓存 `trimLeftBound` / `trimRightBound`
- 整个拖拽过程使用缓存值（siblings 不会在拖拽中变化）

## 四、已修复问题清单

| # | 问题 | 修复方式 |
|---|------|---------|
| 1 | 手柄热区仅 8 pt，难以命中 | `TrimHandleView` 热区扩展至 24 / 32 pt |
| 2 | 附属轨道手柄常驻显示，界面拥挤 | `isSelected` 控制 `isHidden` |
| 3 | 拉伸手柄可越界侵入相邻片段 | `.began` 缓存边界，`clampedRange` 用 `siblingRanges` 约束 |
| 4 | 片段钻进相邻片段底下出不来 | 同上，实时卡位 |
| 5 | 点击附属片段自动弹编辑面板 | `handleTap` 分主/附属轨道处理 |

## 五、验收标准

1. 主轨道片段手柄始终可见，轻触即响应
2. 附属轨道片段未选中时无手柄，点击后弹出加宽手柄
3. 拉伸左手柄：到前一片段右边缘时自动卡位，无法再向左
4. 拉伸右手柄：到后一片段左边缘时自动卡位，无法再向右
5. 无越界、无相互覆盖、无钻底 bug
6. 无相邻片段时，左边界为 0 s，右边界无限制
