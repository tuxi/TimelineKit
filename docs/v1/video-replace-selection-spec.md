# 视频替换智能区间选择弹窗 · 完整需求规范

> 版本：V1 正式规范  
> 优先级：P0（排在现有 P0 任务之后、P1 任务之前）  
> 归档路径：`docs/v1/video-replace-selection-spec.md`

---

## 一、设计总原则

1. 不新增工具栏独立修剪入口，不占用主时间轴拖动手势，不与「片段移动/换位」手势冲突。

2. **永远不裁切、不重新编码、不生成新视频文件**；始终保留相册原始完整视频素材。

3. 所有区间选择，只记录逻辑入点、出点偏移时间，由剪辑器渲染层动态截取播放，物理素材始终是原长完整视频。

4. 图片素材无时间区间概念，直接替换，不走选择弹窗流程。

---

## 二、弹窗触发逻辑

### 替换为图片素材
直接完成替换，不弹出任何选择弹窗；静态图支持无限拉伸，无需时间选区。

### 替换为视频素材

| 条件 | 行为 |
|------|------|
| 新视频原生时长 **等于** 当前片段占用时长 | 直接替换，不弹窗，默认入点 = 0 |
| 新视频原生时长 **大于** 当前片段占用时长 | 自动弹出视频区间选择弹窗 |
| 新视频原生时长 **小于** 当前片段占用时长 | V1 暂不允许替换，友好提示「素材时长不足，请选择更长视频」 |

---

## 三、弹窗内部交互规范

### 页面结构
- **上半部分**：视频实时预览区，跟随选区滑动实时刷新画面。
- **下半部分**：单层简易缩略图时间轴，展示完整原始视频全长。

### 选区规则
- 界面中间固定选区框宽度 = 当前片段固定占用时长，**宽度不可手动缩放**。
- 用户左右滑动底部整条时间轴，选区框位置不动，底层轨道滚动，只改变起始入点。
- 滑动边界锁死：只能在「原始视频完整时长范围内」滑动，禁止滑出首尾越界。

### 确认逻辑
点击确定后不生成新视频、不裁切原文件，仅保存两个字段：
- `clipInTime`：原始视频的起始偏移时间（秒）
- `clipOutTime = clipInTime + 当前片段固定时长`

这两个值对应 `EditorSegment.sourceRange`（`start` = `clipInTime`，`duration` = 片段目标时长）。

---

## 四、核心硬性底层规则（最重要，必须强制遵守）

> ⚠️ **这是整个功能最关键的约束，开发不得以任何形式绕过。**

### 规则 1 — 原始素材永不裁切
替换后工程内始终引用原始完整时长的视频素材，不裁剪、不转码、不生成新片段文件。

### 规则 2 — 渲染层逻辑偏移
剪辑器仅通过 `sourceRange.start`（= `clipInTime`）做逻辑时序偏移，渲染层动态从原视频指定入点开始播放。`CompositionBuilder.srcRange(for:)` 已实现此逻辑，禁止在此之外另行裁切文件。

### 规则 3 — 支持二次编辑
- 随时重新进入区间选择弹窗，修改入点位置；
- 只要原始视频是 10 秒，片段随时可以向外拉伸、重新选区间；
- **不会因裁切变成固定 4 秒而无法二次调整。**

### 规则 4 — 全链路基于原文件
渲染、预览、导出均基于「原完整视频 + 逻辑入点偏移」合成，而非裁切后的短片段。

---

## 五、操作后置效果要求

1. 确认选择后，轨道缩略图自动刷新为当前入点对应的画面（`ThumbnailProvider` 缓存已按 time 分 key，天然支持）。
2. 时间轴预览播放、`UnifiedCompositor` 渲染自动跟随 `sourceRange.start` 偏移。
3. 整次替换 + 选区选择操作，完整支持 Undo / Redo（通过 `EditorStore.mutate` 单次原子提交）。
4. 保留原片段已有的调色、滤镜、转场、目标时长等所有属性，仅替换视频资源与 `sourceRange`。

---

## 六、数据模型映射

```
EditorSegment
├── materialID          → 指向 MaterialsPool 里的原始完整视频 EditorAsset（不变）
├── targetRange         → 片段在时间轴上的位置与时长（不变）
└── sourceRange         → { start: clipInTime, duration: targetRange.duration }
                          ↑ 这是弹窗唯一修改的字段
```

`replaceSegmentMaterial` 在替换时会传入 `nativeDuration`，弹窗确认后调用
`EditorStore.setSourceRange(segmentID:start:)` 单独更新 `sourceRange`，两步操作通过 `mutate` 合并为单个 undo 条目。

---

## 七、为什么必须这样设计

**错误做法**（常见误区）：在弹窗里把原视频裁切成固定 4 秒新文件写入磁盘。

- 用户无法再调整入点
- 无法外延拉伸时长
- 剪辑链路直接锁死，变成一次性死片段
- 导出质量损失（二次编码）

**正确做法**：原素材不变 + `sourceRange` 逻辑入点偏移。

- 永远保留原始全长自由度
- 随时可二次修改选区、拉伸时长
- 零磁盘写入、零质量损失
- 符合 Final Cut Pro / Premiere 等专业剪辑软件的底层设计规范
