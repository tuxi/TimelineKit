# Issue-001 剪辑片段手柄右侧最大时长限制未生效

## 问题归属

| 字段 | 值 |
|------|-----|
| 基线版本 | docs/v1 |
| 所属模块 | 时间轴片段修剪、手柄约束 |
| 状态 | **P1 暂不修复 · 遗留挂起 · 不阻塞 P2 开发** |

---

## 现象

视频类、image_3d、image_motion 带原生素材时长的片段，
右侧手柄仍可向外无限拖动扩大范围，超出素材原始最大时长，超出部分播放黑屏。

---

## 已做修复复盘

1. 已修复手势多次拖拽 UI 漂移、跳变、基准范围陈旧问题（`currentRange` 机制）；
2. 已在 `TimelineImporter` 为 `ai_video` / `image_3d` / `image_motion` 赋值 `nativeDuration`；
3. 已修正 `sourceDurationMap` 静态图清除逻辑，改为以 `nativeDuration` 存在与否作为判断依据（不再依赖 `motionPreset`/`depthEffect` 字段是否为 nil）；
4. `clampedRange` 公式逻辑已具备「起点 + 素材时长」封顶能力，且在 `.changed` 阶段实时执行。

---

## 遗留根因摘要

整条 `sourceDuration` 赋值 → 传递 → 区块读取 → 拖拽实时卡位的端到端链路，
仍存在某一环数据透传/生效时机未对齐，导致部分视频类片段未正确命中右侧边界硬卡位逻辑。

---

## 暂存决策

P1 功能主体已验收完毕，此问题**不阻塞日常剪辑、不阻塞 P2 转场/滤镜/导出开发**，
当前版本暂时搁置修复，纳入 v1 基线已知问题列表，后续迭代再专项深度排查闭环。

---

## 复现步骤

1. 导入本地视频 / ai_video / image_3d / image_motion 片段；
2. 选中片段，拖动片段右侧手柄向外拉长；
3. 可无限超出原始素材时长，播放尾部黑屏。

---

## 预期标准（对齐剪映）

带原生时长的素材片段，右侧手柄拖拽过程中实时硬卡位，禁止超出素材最大时长；
静态图片、字幕、贴纸、文字轨道保持可无限自由拉伸。
